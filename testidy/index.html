Improved School Control

Testidy is a platform that facilitates school control, mainly assistance. For that, QR codes are used in the classrooms, that are scanned by the students to confirm that they are inside the classroom.

In addition to this, other functions that are useful for an educational institution are included, which include schedules, practices, notices, etc. The following are the main characteristics of the project:

  • web application, with features like::
    • separate access for students, teachers and administrators;
    • laboratory registration with automatic generation of QR codes;
    • function for assistance pass during class time;
    • generation of assistance reports in real time;
    • schedules with details to know what class takes place, under what subject, in what place and with what teacher;
    • push notifications to send notices to specific students, groups or the entire school;
    • integrated exams system to perform tests during class time;
    • ... and some extra functions such as separating classrooms, file storage, importing students and teachers through .csv files, among others.
  • android application, with features like:
    • scanning of QR codes to confirm attendance to classes;
    • direct access to the school website;
    • function to request classrooms at any available date and time;
    • history to check received notifications.
Tech wrap-up
Web Application
  • ASP.NET with C #
  • Serves 3 purposes at once:
    • as a web application for students and teachers,
    • as an API for the mobile application,
    • as administrator panel.
  • Connection with Firebase to send push notifications.
  • Integration with jQuery for date selector widget and search filters with autocomplete functions.
  • ASMX Web Services for different functions.
  • SQL Server database.
Mobile Application
  • Native application developed with Android Studio
  • QR code reader integrated within the application.
  • Firebase integration to receive push notifications.
  • Communication with the server through SOAP.
