Surveys System

Suffragium is a system created to make surveys, integrating the use of mobile devices for taking responses combined with powerful analysis and control tools to facilitate the processes and obtain the best results.

It also includes a control of workers to perform audits, with features such as voice recording, limitation of the work area through GPS, verification of the route followed, record of the places where responses were taken, etc.

  • web application, with features like:
    • register workers and administrators;
    • create surveys with questions of different types, selection, open, images;
    • maps with work areas to specify where the interviewers must work;
    • audio recording for each response taken;
    • map with the location of the surveyors in real time and track of their route;
    • consultation of each individual sample, with its audio answers and location;
    • visualizer of reports in the form of graphs, with options to customize the type of the graph, color, among other aspects;
    • generator of personalized queries to obtain data filtered by geographical zones, specific questions, etc;
    • history of actions performed in the system;
  • android application, with features like:
    • administration of the tasks assigned to the worker;
    • capture of surveys, including audio recording;
    • map with the area in which responses must be collected;
    • use of GPS to verify that responses are collected in the assigned area;
Tech wrap-up
Web Application
  • Laravel Web Application.
  • Serves 2 purposes at once:
    • as API for the mobile app,,
    • as administrator panel.
  • Use of Vue.js in the front end for different applications.
  • Integration with Google Maps API to draw and store polygons and locations.
  • REST services for communication with the mobile application.
  • MySQL database.
Mobile Application
  • Native application developed with Android Studio
  • Communication with the server through REST with the Retrofit library.
  • Integration with SQLite to store data of responses taken.
  • Use of GPS to maintain control of locations and tracking in real time.
  • Integration with Google Maps.
